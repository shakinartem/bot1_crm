# PROJECT CONTEXT — BOT 1 CRM

## Что такое БОТ 1

БОТ 1 — CRM / SHARiK Sales Intelligence Bot. Это Telegram-first система для холодных продаж стоматологическим клиникам от имени агентства «ШАРиК digital».

Бот не является простой записной книжкой. Его задача — помогать менеджеру вести базу клиник, фиксировать ЛПР, контакты, касания, статусы, задачи и быстро готовиться к звонку через AI.

## Бизнес-задача

Агентству нужна единая рабочая база лидов, где менеджер видит контекст перед каждым касанием: кто клиника, кто принимает решение, какие контакты есть, что уже обсуждали, какой следующий шаг и чем клиника может быть интересна для digital-продажи.

## Пользователь

Основной пользователь — менеджер по продажам или владелец агентства, который работает с холодной базой стоматологий через Telegram.

## Основные сущности

- `Company` / Clinic — стоматологическая клиника или юрлицо.
- `DecisionMaker` — ЛПР или человек, влияющий на решение.
- `ContactPoint` — телефон, email, сайт, Telegram, WhatsApp, соцсети и другие контакты.
- `LeadInteraction` — звонок, сообщение, встреча, консультация, КП или заметка.
- `FollowUpTask` — задача менеджера на следующий контакт.
- `CallRecord` — запись звонка и будущая транскрибация.

## Связь с будущей сетью ботов

БОТ 1 — первый модуль будущей сети:

- БОТ 1: CRM / поиск клиентов / продажи / договоры.
- БОТ 2: Consultation AI / подготовка консультаций и аудитов.
- БОТ 3: Content Generator / контент-планы, посты, статьи, видео.
- БОТ 4: Autoposter / публикация в соцсети.
- БОТ 5: Reporter / отчеты, охваты, планы работ.

БОТ 1 должен уметь передавать подготовленного клиента дальше через API, events или export-слой без привязки бизнес-логики к Telegram.

## Архитектурные принципы

- Telegram-first, но backend не должен зависеть от Telegram.
- Бизнес-логика живет в service-слоях.
- API и Telegram-бот используют одни и те же сервисы.
- База данных должна быть расширяемой и SQLite/PostgreSQL-ready.
- Все функции проектируются с учетом будущего web UI / Telegram Mini App.
- AI-провайдер заменяемый: OpenRouter, Ollama или другой провайдер через общий интерфейс.
- Изменения делаются маленькими совместимыми шагами: сначала совместимый слой, потом рефакторинг.
